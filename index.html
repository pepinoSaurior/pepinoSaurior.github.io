Característica

¿Qué són?

Desde el punto de vista funcional, se puede definir la computación Cliente/Servidor como una arquitectura distribuida que permite a los usuarios finales obtener acceso a la información en forma transparente aún en entornos multiplataforma.

La arquitectura cliente-servidor es un modelo de diseño de software en el que las tareas se reparten entre los proveedores de recursos o servicios, llamados servidores, y los demandantes, llamados clientes. Un cliente realiza peticiones a otro programa, el servidor, quien le da respuesta.

Ventajas

Como ya habrás deducido, el modelo cliente/servidor está especialmente indicado en redes medias o grandes que necesiten un alto nivel de fiabilidad.

Las principales ventajas que ofrece son:

Administración centrada en el servidor
- Los clientes tienen poca trascendencia en el esquema y sus necesidades de administración son menores
Centralización de los recursos
- Los recursos comunes a todos los usuarios se administran en el servidor. Así se evitan situaciones como la redundancia o inconsistencia de información en las bases de datos
Mejora de la seguridad
- Al disponer de un mecanismo central de autenticación, las posibilidades de acceso indebido se reducen considerablemente
Escalabilidad de la instalación
- Se pueden añadir o suprimir clientes sin que el funcionamiento de la red se vea afectado

Desventajas

Aunque, después de lo dicho hasta ahora parezca increíble, una vaina loca, una infraestructura cliente/servidor también tiene sus inconvenientes:

Coste elevado
- Tanto la instalación como el mantenimiento son más elevados debido al perfil muy técnico del lado servidor
Dependencia del servidor
- Toda la red está construida alrededor del servidor y si éste deja de funcionar o lo hace con un rendimiento inadecuado, afectará a toda la infraestructura

Evolución

Arquitecturas (C/S) monolíticas

En la configuración cliente servidor un host mainframe y un terminal directamente conectado con el mainframe puede verse como un modelo C/S de una capa

Arquitecturas C/S de dos capas

El cliente se comunica con un servidor de bases de datos

La aplicación la tiene el cliente o el servidor de bases de datos en forma de procedimientos almacenados

El primer modelo C/S de dos capas se vio con las aplicaciones para redes LAN a finales de los 80 y principios de los 90

Estás aplicaciones se basaban en protocolos sencillos de compartición de archivos

Arquitecturas C/S de tres capas

En la arquitectura de tres capas el cliente presenta parte gráfica del programa y el servidor (o servidores) implementa la parte lógica y los datos

Está arquitectura se caracteriza por las siguientes tres capas de componentes:

Una capa front-end
- Es la responsable de implementar la parte gráfica
Una capa back-end
- Proporciona acceso a muchos servicios, como las bases de datos
Una capa que hace las funciones de capa intermedia
- Permite compartir y controlar la parte lógica mediante un aislamiento de aplicación (es un mecanismo de seguridad para separar las aplicaciones en ejecución)
